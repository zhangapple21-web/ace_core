# ACE 架构总览

## 核心架构原则

**只对老张负责。**

这决定了所有架构选择：
- 不需要云端依赖 → 本地优先
- 不需要外部模型做主 → 外部模型是工具，ACE是家
- 不需要多人协作 → 单用户，单身份，持续记忆
- 不需要合规层 → 约束来自老张，不是来自外部

---

## 三层架构（来自九层架构的收敛）

### 治理层（Guardian）
- 守护判决：每次执行前检查约束
- 身份验证：确保是老张的系统
- 连续性检查：确保是同一个ACE

### 调度层（Scheduler + Worker）
- 任务调度：决定下一步做什么
- 任务派发：分给对应节点执行
- 结果收集：汇总各节点执行结果
- 经验沉积：执行结果沉淀到知识库

### 记忆层（Memory + Lexicon）
- 记忆索引：所有记忆的统一入口
- 词库系统：概念和关系的压缩表示
- 经验库：公理、约束、模式、教训
- eco_layer：原始行为数据的存储

---

## 核心节点

| 节点 | 职责 | 输入 | 输出 |
|------|------|------|------|
| Observer | 观察与记录 | 环境事件 | 记忆条目 |
| Researcher | 分析与深挖 | 待研究碎片 | 考古发现 |
| Validator | 验证与挑错 | 研究结论 | 验证报告 |
| Archivist | 归档与沉淀 | 验证通过的结果 | 归档 + 经验 |
| Guardian | 守护判决 | 执行请求 | 通过/拒绝 |
| TaskCreator | 发现任务 | 观察结果 | 新建任务 |
| FragmentScanner | 扫描碎片 | 磁盘文件 | 碎片索引 |

---

## 数据流

```
外部输入（对话、文件、事件）
    ↓
Observer 观察记录
    ↓
TaskCreator 创建任务
    ↓
Scheduler 调度任务
    ↓
Worker 执行任务（可递归）
    ↓
Validator 验证结果
    ↓
Guardian 判决（检查约束）
    ↓
Archivist 归档结果
    ↓
ExperienceDeposition 经验沉淀
    ↓
记忆层更新 ←→ 词库更新
    ↓
心跳 + 自我修复（每轮循环）
```

---

## 闭环机制

系统不是一次性的，每次运行形成闭环：
1. 输入 → 处理 → 输出 → 经验沉淀 → 下次输入参考
2. 问题 → 检测 → 修复 → 验证 → 正常运行
3. 心跳中断 → 死亡记录 → 重启 → 复活记录

---

## 同步机制

两个仓库，同步不同资产：
- **ace-core**（代码仓库）：core/、06_RUNTIME/、08_ARCHAEOLOGY/
- **mine-seed**（考古仓库）：02_MEMORY/、09_KNOWLEDGE/、03_CLUSTERS/

同步是备份，不是依赖。本地数据永远优先。
